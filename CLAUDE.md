# 교대근무 루틴 앱 (CLAUDE.md)

## 프로젝트 개요
교대근무자를 위한 루틴 관리 PWA. 기존 루틴/습관 앱은 '요일(월~일)' 기반이라
교대근무자에게 안 맞는다. 이 앱의 핵심 차별점은 요일이 아니라
'교대 주기의 Day'에 루틴을 연결하는 것이다.
- 기존 교대 앱 = '언제 일하는지'만 보여줌
- 일반 루틴 앱 = '오늘 뭘 할지' 챙기지만 요일 기반
- 이 앱 = 교대 주기에 루틴을 얹음 (이 틈만 깊게 판다)

## 배포
- GitHub Pages. 저장소 routine-app.
- 배포 = git add → commit → push. ("배포해줘"로 통칭)
- 접속 주소: https://gmdwn916-cmd.github.io/routine-app/
- 데이터는 사용자 기기 브라우저(localStorage)에만 저장되며,
  코드(index.html)는 push로 갱신된다. 코드 배포는 사용자 데이터에 영향 없음.

## 파일 구조
- index.html 하나에 HTML/CSS/JS 모두 포함. 단일 파일 유지.
- 외부 라이브러리 없음. 순수 HTML/CSS/JS.

## 용어 (앱 전체에서 이 정의로 통일. 절대 혼동 금지)
- **루틴** = 주기 반복. Day(D1~DN) 기준. 예: 주짓수, 미리 요리.
  같은 Day면 다음 주기에도 반복된다.
- **할 일** = 일회성. 실제 날짜 기준, 그날 하루만. 예: 머리 자르기,
  커튼 세탁, 괌 여행. 내부적으로 events에 저장한다.
- "일정"이라는 표현은 쓰지 않는다. 모두 '할 일'로 통일.

## 작업 방식 (반드시 지킬 것)
1. 코드를 수정하기 전에 항상 구현 계획을 먼저 제시하고,
   사용자 확인을 받은 뒤에만 수정한다. 확인 전 코드 작성 금지.
2. 수정 대상 파일은 그 작업에서 명시된 파일로만 한정한다.
   명시되지 않은 파일은 절대 건드리지 않는다.
3. 데이터 구조를 바꿀 때는 기존 localStorage 데이터를 보존한다.
   마이그레이션 코드를 넣고, 데이터 유실을 절대 만들지 않는다.
4. 수정 후에는 스스로 코드를 다시 읽고 점검한 뒤,
   무엇을 점검했는지 한국어로 요약 보고한다.
5. 모든 응답과 설명은 한국어로 한다.

## 데이터 모델 (localStorage key: shiftRoutine)
- cycleDays: 주기 길이(정수, 상한 없음). 보통 N교대 × 7.
- baseDate: "YYYY-MM-DD" (Day 1에 해당하는 날짜)
- days[]: 각 Day 객체 (기본 패턴, 설정탭이 단일 원본)
  - dayIndex
  - shiftType: { name }   // 추후 hourlyRate, hours 확장 대비해 객체로 유지
  - items[]: 루틴 항목 배열
    - text
    - category   // 운동/식단/청소/준비/기타 중 하나, 없으면 '기타'
  - done   // 자리만 존재, 현재 UI 미노출
- events[]: 할 일(일회성). { date: "YYYY-MM-DD", text }
- shiftOverrides: 근무 덮어쓰기 (기본 패턴은 안 건드리고 위에 덮어씀)
  - 날짜 예외(이번 날짜만): { date: "YYYY-MM-DD", shiftName }
  - D번호 예외(앞으로 D○마다): { dayIndex, shiftName }
- shiftColors: 근무유형 이름 → 색 매핑. 예: { "주": "#cfe3ff", ... }
- weekStart: 주 시작 요일. 0 = 월요일, 1 = 일요일 (기본 0)
- skips[]: 오늘 스킵한 루틴 기록. { date, text } (루틴 원본 미수정)

## 핵심 로직 (건드릴 때 주의)
- 날짜 차이는 반드시 로컬 자정 기준으로 계산한다.
  "YYYY-MM-DD"를 split해서 new Date(년, 월-1, 일)로 생성.
  new Date(문자열)을 직접 쓰지 말 것 (UTC 파싱으로 한국 시간대에서 하루 밀림).
  diff = Math.round((todayMid - baseMid) / 86400000)
  dayIndex = ((diff % cycleDays) + cycleDays) % cycleDays + 1
- 근무 표시 우선순위: 날짜 예외 > D번호 예외 > 기본 패턴.
  **설정탭의 기본 근무 패턴은 어떤 근무 수정으로도 바뀌지 않는다.**
  근무 변경 시 "이번 날짜만" vs "앞으로 D○마다"를 물어 각각 예외로 저장.
- 월 달력: weekStart 설정에 따라 월요일/일요일 시작. 요일 헤더·칸 배치 반영.
  Day 1을 baseDate 실제 요일 칸부터 전개. 주기가 7의 배수 아니면 마지막 줄이
  비는 것은 정상.
- 카테고리 색·근무 색은 코드 상단 상수에서 관리. 한 곳만 고치면 전체 반영.
- 루틴 표시와 할 일 표시는 시각적으로 구분한다(섞이지 않게).

## 화면 구성 (하단 탭 4개)
- 루틴: 루틴을 한 곳에서 관리(같은 이름=같은 루틴으로 그룹핑, 방법 A).
  새 루틴 만들기 → 달력에서 날짜 복수 선택 → 해당 Day에 추가.
- 오늘: 날짜·요일(가장 크게) > 근무 > D번호(작게, 오른쪽 구석) 순.
  오늘의 루틴·할 일 표시. 루틴 항목 스킵 → 다른 날 할 일로 넘기기 가능.
- 달력: 월 달력. 근무유형 색으로 칸 구분. 날짜 탭 → 상세(루틴/할 일 추가).
  좌우 스와이프로 월 이동.
- 설정: 교대 수(×7=주기)·근무 패턴·시작 날짜·근무 색·주 시작 요일.
  설정탭이 기본 패턴의 단일 원본.
