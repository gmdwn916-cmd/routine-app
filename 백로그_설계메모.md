# 교대근무 루틴 앱 — 보관 설계 메모 (백로그)

작성 시점 기준, 흥주와 미리 설계해둔 기능 모음.
대부분 "실사용 후 필요하면 만들기"로 판단 보류 상태.
각 메모는 그대로 프롬프트로 다듬어 클로드코드에 넘길 수 있음.

---

## 용어 (앱 전체 통일 — 절대 혼동 금지)
- **루틴** = 주기 반복(Day 기준). 예: 주짓수
- **할 일** = 일회성(날짜 기준, 하루). 예: 괌 여행. 내부적으로 events.
- **리마인더** = 날짜/주기는 있지만 근무 보고 조정. 예: 생일, 머리 자르기.
  (임시 이름, 실사용 후 변경 가능)

---

## 지금 바로 만들 수 있는 것

### ★ 스킵 (프롬프트 완성 상태)
오늘 못 하는 루틴 항목을 '스킵'하고, 원하면 다른 날 '할 일'로 넘김.
- 오늘 탭 루틴 항목에 스킵 액션.
- 스킵 시: 오늘만 넘김 표시(루틴 원본 안 건드림, 다음 주기 그대로)
  + "다른 날 할 일로 옮길까요?" → 날짜 선택 → events에 추가.
- 데이터: skips[] { date, text } / events에 추가. 기존 보존.

## 실사용 후 만들 것 (의존 순서대로)

### 1) 루틴 체크(완료 기록) — 성취율의 선행 조건
- 오늘 탭 루틴 항목의 '완료' 체크(이름 텍스트 말고 체크 영역 별도).
- 날짜+항목 기준 완료 기록: done_log[] { date, text }
- 완료 / 스킵 / 미처리 구분. 루틴 원본(items) 수정 안 함.

### 2) 항목별 성취율 — 1) 다음
- 각 루틴의 '예정일 대비 완료율'. 예: 주짓수 예정 8일 중 7일 = 88%.
- 기간 선택(이번 주기/이번 달). 기록에서 계산(저장 안 함).

### 3) 루틴 이름 어디서나 탭 → 관리 화면 이동 — 1),2) 다음
- 루틴 이름을 어디서 보든(오늘/달력 상세) 누르면 그 루틴 관리 화면으로.
- 관리 화면 = 루틴 탭의 관리 화면 재사용
  (들어간 Day 목록 / Day 추가·빼기 / 성취율 / 이름·카테고리 / 삭제).
- 동작 분리: 이름 탭=이동 / 체크=완료. 겹치지 않게.

### 4) 리마인더 — 독립적, 실사용 후
- 날짜/주기는 있지만 근무 보고 조정하는 항목.
- 두 유형: 날짜 고정형(생일 매년 8/15) / 간격형(머리 6주).
- 때 오면 "챙길 때 됐어요" 신호(날짜 강제 X) → 사용자가 근무 보고
  '할 일'로 잡음 → 완료 시 재카운트/내년.
- 데이터: reminders[] { name, type, targetDate?, intervalDays?, lastDoneDate? }

### 5) 여러 팀 근무 보기 — 1순위 킬러 후보 (사업 확장 시)
- 같은 직장 여러 팀이 각각 무슨 근무인지 한눈에(업무 목적).
- 오프셋 구조: 모든 팀이 같은 패턴을 시차만 두고 돎.
- 설정 "팀 근무 추가": 교대 수만큼 줄 자동(내 팀 포함, 내 팀 고정).
  나머지 팀은 팀 이름 입력 + 화살표(◀▶)로 근무 칸을 오늘 실제에 맞춤 → 저장.
- 전체 팀 근무표 화면: 어느 날짜든 전 팀 근무 한눈에.
- 데이터: teams[] { name, offsetDays }. 패턴은 내 설정 공유.

### 6) 오늘 통합 뷰 — '오늘 탭 완성형' (범용 투두 아님)
- 앱 안의 루틴 + 할 일 + 리마인더 신호를 오늘 하루로 모아 보기.
- 주의: 외부(이메일·업무)까지 끌어와 만능 투두가 되지 말 것.

## 접어둔 것 (당분간 안 만듦)
- **자체 알람**: PWA 알람은 약함(폰/브라우저 닫으면 안 울림).
  만들려면 자체 알람 대신 폰 기본 캘린더/알람에 '얹는' 방식 검토.
  (2번 논의 미완 — 흥주가 알람으로 원하는 게 소리인지 눈에 띄는 표시인지 확인 필요)
- **시간 배치(타임블로킹)**: 전문 앱(Structured) 영역. 레드오션.
- **일반 근무자 확장**: 차별점(교대근무자) 흐림. 루티너리(500만)와 정면 승부됨.
  교대근무자는 대안이 없어 꽉 잡을 수 있는 시장 → 여기 집중.
